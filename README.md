# منصة تقييم العملاء — GitHub + Firebase

مشروع جاهز لواجهة تقييم عملاء احترافية باللغة العربية، مع:

- صفحة عامة لعرض بيانات الشركة واللوجو ووسائل التواصل والموقع الجغرافي
- نموذج تقييم للبضاعة والخدمة والملاحظات
- لوحة إدارة مخفية تظهر بعد الضغط على اللوجو **6 مرات خلال 5 ثوانٍ**
- تسجيل دخول المدير عبر **Firebase Auth**
- التحقق من **UID الأدمن** بحيث لا تظهر لوحة الإدارة إلا للمستخدم:
  - `JxKXouwjdadht4wSMPf1qtbeW9n1`
- حفظ بيانات الشركة في **Firestore**
- حفظ ردود العملاء في **Firestore**

## بيانات Firebase المدمجة
تم تضمين الـ config والـ admin UID داخل `public/app.js` كما طلبت.

## تشغيل المشروع
1. ارفع الملفات إلى GitHub.
2. من Firebase Console فعّل:
   - Authentication > Email/Password
   - Firestore Database
3. تأكد أن حساب الأدمن الموجود في Firebase Auth يملك UID التالي:
   - `JxKXouwjdadht4wSMPf1qtbeW9n1`
4. ثبّت Firebase CLI إن لم يكن مثبتًا:
   ```bash
   npm install -g firebase-tools
   ```
5. سجّل الدخول:
   ```bash
   firebase login
   ```
6. من داخل مجلد المشروع نفّذ:
   ```bash
   firebase deploy
   ```

## فتح لوحة الإدارة
- افتح الموقع
- اضغط على اللوجو **6 مرات بسرعة**
- ستظهر نافذة تسجيل دخول الأدمن
- بعد الدخول الصحيح ونجاح التحقق من الـ UID ستظهر لوحة التحكم

## تعديل بيانات الشركة
من لوحة الإدارة يمكنك تعديل:
- اسم الشركة
- الوصف المختصر
- العنوان
- رابط الخريطة
- ساعات العمل
- أرقام التواصل
- البريد الإلكتروني
- واتساب
- رابط الموقع الرسمي
- روابط السوشيال ميديا
- المواسم
- الشعار

## الملفات المهمة
- `public/index.html` الواجهة
- `public/styles.css` التنسيق
- `public/app.js` المنطق البرمجي + Firebase
- `firestore.rules` قواعد Firestore
- `firebase.json` إعدادات النشر
- `.firebaserc` ربط المشروع بـ Firebase

## ملاحظات
- المشروع static ويعمل مباشرة على Firebase Hosting.
- الشعار يُدار عبر رابط مباشر داخل Firestore بدون استخدام Firebase Storage.
- إذا أردت توسيع لوحة التحكم لاحقًا يمكن إضافة فلاتر أكثر أو تصدير Excel أو رسوم بيانية تفصيلية.
